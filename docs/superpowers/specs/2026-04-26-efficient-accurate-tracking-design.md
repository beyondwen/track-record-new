# 省电且准确的轨迹采集方案设计

## 背景

当前轨迹采集链路的主要矛盾不是单一参数过紧，而是定位、实时过滤、后处理和渲染多层都在收缩。对于用户当前目标，重点不是"任何场景都不断线"，而是：

- 静止时尽量省电。
- 室外移动时尽快拿到可信轨迹。
- 室内没有精确点位时不记录。
- 恢复记录前必须确认连续好点，避免误接和乱画。

## 目标

构建一套以"静止低功耗、室外移动可信、室内不乱记"为核心的分阶段采样方案。

## 非目标

- 不解决室内连续精确轨迹问题。
- 不做 Wi‑Fi 指纹、蓝牙 Beacon、UWB 等室内定位能力。
- 不为了轨迹连续性而补假点、插值或粗暴连线。
- 不追求纯 GPS 模式。

## 用户确认的约束

- 采用折中方案：静止时尽量省电，移动时逐步变准。
- 室内无精确点位时不记录。
- 恢复记录前，必须连续拿到 2～3 个好点。
- 恢复后新开一段，不强接室内空档。
- 允许轨迹在室内、遮挡区、弱信号区断开。

## 方案概述

推荐采用"三阶段状态机 + 连续好点确认 + 室内不记录"方案。

核心思路如下：

1. 在待机状态下，使用 `PASSIVE_PROVIDER`、Activity Recognition 和轻量传感器作为低功耗唤醒信号。
2. 检测到疑似移动后，先进入观察态，只做低成本确认，不立即高频记录。
3. 只有在连续拿到 2～3 个精确、连续、可信的好点后，才进入正式记录态。
4. 一旦失去可信定位，则停止落点，进入失联观察态；恢复后必须重新累计好点，并新开一段。

## 状态机设计

### 1. `IDLE`

低功耗待机态。

职责：

- 监听 `PASSIVE_PROVIDER`。
- 监听 Activity Recognition。
- 监听轻量级运动传感器变化。
- 可选 very low frequency 的粗定位确认。
- 不进入正式轨迹记录。

目标：

- 尽量不主动拉起高成本定位。
- 只判断"是否可能开始移动"。

### 2. `SUSPECT_MOVING`

疑似移动观察态。

触发来源：

- 传感器显示持续移动。
- `PASSIVE_PROVIDER` 或低成本位置出现新鲜位移。

职责：

- 观察是否存在连续位置变化。
- 判断位置精度是否达标。
- 累计连续好点数量。
- 不正式落点，不生成轨迹段。

目标：

- 用低成本方式确认"人真的在移动且当前环境可记录"。
- 避免一有移动迹象就立刻进入高成本记录。

### 3. `ACTIVE_RECORDING`

正式记录态。

进入条件：

- 连续 2～3 个好点达标。
- 点间位移合理，不是原地抖动。
- 时间连续且足够新鲜。

职责：

- 开启较积极的定位采样。
- 正式落点、分段、连线。
- 仅过滤明显脏点，不做过度实时删点。

目标：

- 在室外移动场景中尽量保留真实轨迹细节。

### 4. `SIGNAL_LOST`

移动中失去可信定位的观察态。

触发条件：

- 连续一段时间拿不到好点。
- 定位精度持续恶化。
- GPS 在室内或遮挡区不可用。

职责：

- 停止落点。
- 不补点，不插值，不粗连线。
- 等待重新累计 2～3 个好点。

目标：

- 宁可断开，也不乱记。
- 在不可信环境下及时止损。

## 状态切换规则

### `IDLE -> SUSPECT_MOVING`

满足任一类触发即可进入观察态：

- Activity Recognition 判定为 walking、on_bicycle 或 in_vehicle。
- 加速度在短窗口内持续活跃。
- `PASSIVE_PROVIDER` 或低成本位置出现新鲜位移。

说明：

- 此时只说明"疑似开始动了"。
- 不代表已经具备正式记录条件。

### `SUSPECT_MOVING -> ACTIVE_RECORDING`

必须同时满足：

- 连续拿到 2～3 个新鲜位置点。
- 这些点的精度都达标。
- 点间位移合理，不是原地抖动或单个跳点。
- 时间间隔合理，不是旧缓存点批量冒出。

说明：

- 只有确认"真在动且当前定位可信"后，才开始正式记录。

### `SUSPECT_MOVING -> IDLE`

当以下情况发生时回退待机：

- 传感器不再显示移动。
- 没有持续位置变化。
- 观察窗口内一直拿不到连续好点。

### `ACTIVE_RECORDING -> SIGNAL_LOST`

满足任一条件进入失联观察态：

- 连续一段时间没有精确点。
- 精度持续恶化。
- 位置更新明显中断。
- 当前环境推断为室内或遮挡严重。

### `SIGNAL_LOST -> ACTIVE_RECORDING`

恢复条件与首次进入记录一致：

- 再次连续拿到 2～3 个好点。
- 确认不是抖动或跳点。
- 恢复后新开一段，不强接中间空档。

### `ACTIVE_RECORDING / SIGNAL_LOST -> IDLE`

当以下条件成立时结束本次活动：

- 传感器持续静止。
- 位置不再发生有效变化。
- 长时间没有恢复出可信移动。

## 分阶段采样策略

### `IDLE`

- 常驻 `PASSIVE_PROVIDER`。
- 常驻 Activity Recognition。
- 轻量使用运动传感器。
- 不开启高频 GPS。
- 可选 very low frequency 的粗定位确认，但不是必须。

### `SUSPECT_MOVING`

- 先不进入最高成本定位模式。
- 优先利用 `PASSIVE_PROVIDER`、低成本位置和传感器确认移动趋势。
- 开短观察窗口，累计连续好点。
- 只在确认条件达成后才升级。

### `ACTIVE_RECORDING`

- 进入更积极的定位采样。
- 优先 GPS，但不强制纯 GPS。
- network 仅作辅助，不主导正式落点。
- 实时过滤只拦截明显异常点。

### `SIGNAL_LOST`

- 降低采样强度。
- 停止高频硬拉精确定位。
- 保留观察能力，等待环境恢复。
- 恢复不了则最终回到 `IDLE`。

## 好点判定规则

一个点进入"可记录候选"至少要满足以下条件：

### 1. 新鲜

- 不是很久以前的缓存点。
- 时间戳接近当前时刻。
- 不允许用旧点误判为新一轮移动。

### 2. 精度达标

- 精度在可接受范围内。
- 室内常见的大精度漂移点直接淘汰。

### 3. 位移真实

- 与前一个候选点之间存在合理位移。
- 过小视为抖动。
- 过大视为异常跳点候选。

### 4. 连续性成立

- 不是单个好点就直接恢复。
- 必须连续 2～3 个都满足条件。

## 室内与弱信号策略

### 室内规则

- 室内没有精确点位时不记录。
- 传感器检测到移动也只进入 `SUSPECT_MOVING` 或 `SIGNAL_LOST`。
- 如果始终没有连续好点，不生成轨迹点。

### 弱信号规则

- 不使用粗定位硬连线。
- 不补点，不插值，不猜路线。
- 重新拿到连续好点后，再恢复正式记录。

## 分段与展示规则

### 分段

- 室内、地下或遮挡区造成的空档直接断开。
- 恢复记录时新开一个 segment。
- 不强接上一段的结尾。

### 展示

- 地图只画可信点形成的轨迹。
- 不使用不可信点拼出"连续感"。

### 统计

- 距离、时长只基于可信记录计算。
- 室内无精确点位的时段不做路线估算。

## 方案取舍

### 收益

- 静止时功耗明显下降。
- 室外移动时轨迹可信度提高。
- 室内不会出现明显漂移折线。
- 恢复记录更稳，不容易误接。

### 代价

- 起步前几秒可能少一点细节。
- 室内短移动会直接断开。
- 总里程会偏保守，因为不估算室内无效区间。

## 验证标准

### 1. 静止功耗下降

- 手机长时间静止时，不应频繁进入高成本定位。
- `IDLE` 应成为常态。

### 2. 室外移动恢复足够快

- 从开始走路到进入正式记录，延迟可接受。
- 恢复前能稳定拿到连续 2～3 个好点。

### 3. 室内不乱记

- 室内短移动不生成大量漂移点。
- 不出现错误连线。
- 允许轨迹自然断开。

### 4. 出楼后稳定恢复

- 从室内回到室外后，能重新累计好点。
- 恢复后新开一段，不强接室内空档。

### 5. 总体轨迹可信度提升

- 历史轨迹更接近真实室外路线。
- 被去掉的是不可信片段，而不是正常室外路径。

## 推荐测试场景

### 场景 A：室外短走

- 出门走 3～5 分钟。
- 验证能否较快进入正式记录。

### 场景 B：室外 -> 室内 -> 室外

- 进楼 2～3 分钟再出来。
- 预期：室外有轨迹，室内断开，出楼后恢复并新开一段。

### 场景 C：长时间静止

- 手机放桌上 30 分钟以上。
- 验证几乎不产生无效记录。

### 场景 D：室内走动但无好定位

- 办公楼内来回走动。
- 预期：传感器可能检测到移动，但不正式记录轨迹。

## 最终结论

推荐落地方案为：

- 采用三阶段状态机。
- 使用 `PASSIVE_PROVIDER`、Activity Recognition 和传感器作为低功耗唤醒信号。
- 正式记录前要求连续 2～3 个好点。
- 室内无精确点位时不记录。
- 恢复记录时新开一段，不补中间空档。

这套方案优先保证轨迹可信度，其次兼顾续航，不以"任何场景不断线"为目标。
