## 1. 背景

当前项目已经具备以下能力：

- 在手机端基于窗口特征、轻量模型和平滑器输出 `START / STOP / HOLD`。
- 将自动决策事件落库，并支持历史页反馈。
- 支持导出决策事件样本并在外部训练脚本中继续处理。

但当前训练数据仍然存在两个核心问题：

- 自动记录链路尚未达到可直接作为高质量标签来源的稳定程度。
- 现有导出样本主要围绕自动决策事件组织，不适合直接表达「手动开始 / 手动结束」这种高置信边界标签。

用户当前希望先停用自动记录入口，转为完全手动控制开始和结束，通过人工操作构造一批高质量边界样本，待模型训练成熟后，再由模型接管开始 / 结束决策。

## 2. 设计目标

### 2.1 目标

- 过渡期内停用自动记录入口和运行时自动触发逻辑。
- 仅保留手动开始、手动结束作为记录控制方式。
- 在现有记录模型中显式记录开始来源、结束来源和对应手动边界时间。
- 只将「手动开始 + 手动结束」的完整记录段纳入训练样本来源。
- 让训练导出可以基于手动边界窗口生成 `start / stop` 的高质量正样本。

### 2.2 非目标

- 本设计不在手机端引入在线训练或增量训练。
- 本设计不彻底删除自动记录相关底层代码和数据库结构。
- 本设计不立即重建现有模型结构或训练算法。
- 本设计不要求第一阶段就生成完整负样本。

## 3. 已确认的用户偏好

本次设计明确采用以下用户偏好：

- 过渡期先停用自动记录，但底层代码先保留。
- 训练样本只使用「手动开始 + 手动结束」的完整记录段。
- `start` 正样本窗口：手动开始前 30 s 到后 60 s。
- `stop` 正样本窗口：手动结束前 30 s 到后 60 s。
- 旧的自动记录历史数据继续展示，但训练导出时一律忽略。
- 来源字段直接落在现有记录模型中，不单独新增训练元数据表。

## 4. 总体方案

本设计采用「运行时停用自动记录 + 数据层补充手动边界来源 + 导出侧按边界窗口组织训练样本」的过渡方案。

整体思路如下：

1. 前台与后台不再自动开始或自动结束记录。
2. 用户手动控制每段记录的开始与结束。
3. 每段记录在存储层补充：
   - `startSource`
   - `stopSource`
   - `manualStartAt`
   - `manualStopAt`
4. 训练导出只选择 `startSource = MANUAL` 且 `stopSource = MANUAL` 的完整记录段。
5. 导出结果携带手动边界时间，后续由导出 / 训练脚本展开成 `start / stop` 正样本窗口。

## 5. 运行时变更

### 5.1 自动记录入口停用

过渡期内，应用不再向用户暴露自动记录入口，包括但不限于：

- 智能记录 / 自动记录开关
- 自动记录状态提示
- 用于重新加载自动决策模型的用户入口

用户看到的记录控制方式应收敛为纯手动。

### 5.2 自动运行链路停用

过渡期内不再允许以下行为发生：

- 自动启动后台智能记录服务
- 根据现有决策模型自动开始记录
- 根据现有决策模型自动结束记录

此处的「停用」指运行时不再使用，而不是物理删除代码。自动记录相关底层代码、模型代码、诊断代码和存储结构先保留，以降低后续接回模型决策的成本。

### 5.3 旧数据兼容

历史中已有的自动记录数据继续正常显示，不做删除、不做隐藏。

但这些数据在训练导出阶段一律忽略，不进入新一轮训练集。

## 6. 数据模型变更

### 6.1 来源字段

在现有记录数据模型中补充以下字段：

- `startSource`
- `stopSource`

建议来源枚举至少包含：

- `MANUAL`
- `AUTO`
- `UNKNOWN`

其中：

- 过渡期内新产生的记录应主要为 `MANUAL`
- 历史旧数据可根据现有来源能力回填为 `AUTO` 或 `UNKNOWN`

### 6.2 手动边界时间

在现有记录数据模型中补充以下字段：

- `manualStartAt`
- `manualStopAt`

语义定义如下：

- `manualStartAt`
  当一次记录由用户手动开始时，记录该操作时间戳。
- `manualStopAt`
  当一次记录由用户手动结束时，记录该操作时间戳。

若某一端不是手动触发，则对应字段为空。

### 6.3 训练资格判定

一段记录只有同时满足以下条件，才具备训练资格：

- `startSource = MANUAL`
- `stopSource = MANUAL`
- `manualStartAt != null`
- `manualStopAt != null`
- 记录段本身完整、未损坏、未被用户取消

否则一律不进入训练导出。

## 7. 样本定义

### 7.1 基本原则

本阶段不再将整段「记录中 / 非记录中」状态直接映射为正负样本，而是只围绕手动边界构造高质量事件样本。

原因如下：

- 当前目标是判断「何时开始 / 何时结束」，而不是整段状态分类。
- 整段打标签会让大量中间状态淹没真正的边界时刻。
- 边界附近更接近用户真实决策意图。

### 7.2 `start` 正样本窗口

对于满足训练资格的记录段：

- 以 `manualStartAt` 为锚点
- 取窗口 `[-30 s, +60 s]`

该窗口内样本视为 `start` 的正样本候选。

### 7.3 `stop` 正样本窗口

对于满足训练资格的记录段：

- 以 `manualStopAt` 为锚点
- 取窗口 `[-30 s, +60 s]`

该窗口内样本视为 `stop` 的正样本候选。

### 7.4 窗口外处理

第一阶段不对窗口外样本强行定义负标签。

处理原则：

- 模糊区先丢弃
- 不为追求样本数量牺牲标签纯度
- 等第一批模型训练和误差分析完成后，再决定是否补充负样本规则

## 8. 导出策略

### 8.1 导出目标

导出链路的目标不是在 App 内直接完成训练样本成型，而是导出足够稳定的手动边界元信息，供训练侧进行窗口展开。

### 8.2 导出内容

对于具备训练资格的记录段，导出至少应包含：

- 记录段唯一标识
- 记录开始时间、结束时间
- `startSource`
- `stopSource`
- `manualStartAt`
- `manualStopAt`
- 与该记录段相关的轨迹点或窗口特征来源

### 8.3 与现有导出能力的关系

现有 [`TrainingSampleExporter.kt`](/Users/a555/StudioProjects/track-record-new/app/src/main/java/com/wenhao/record/data/tracking/TrainingSampleExporter.kt) 主要面向自动决策事件导出。

本设计建议新增或扩展一条「手动闭环记录段导出」路径，但不强制删除现有自动决策事件导出能力。

两条导出链路可以并存：

- 旧链路：自动决策事件分析
- 新链路：手动边界训练样本来源

## 9. 组件职责调整

### 9.1 前台 UI

前台 UI 负责：

- 只暴露手动开始 / 手动结束控制
- 不再展示自动记录入口
- 不再引导用户依赖当前自动决策结果

### 9.2 后台服务

后台服务在过渡期的职责调整为：

- 不参与自动开始 / 自动结束
- 仅保留必要的手动记录支持逻辑
- 对旧自动记录运行逻辑采取停用而非删除策略

### 9.3 训练导出模块

训练导出模块负责：

- 过滤掉旧自动记录数据
- 只导出双手动闭环记录段
- 输出手动边界时间和训练资格元信息

## 10. 迁移与兼容策略

### 10.1 旧数据

旧自动记录数据：

- 继续在历史页正常显示
- 不参与第一阶段训练导出
- 不需要立即回填为手动边界样本

### 10.2 新数据

新产生的手动记录：

- 必须尽量完整记录来源字段和手动边界时间
- 成为第一阶段训练集的唯一高质量来源

### 10.3 回退能力

由于自动记录相关代码和结构仍然保留，后续如需重新接入模型决策，应能在以下前提下低成本恢复：

- 恢复 UI 入口
- 恢复后台服务自动触发
- 恢复训练导出中对自动事件的附加分析

## 11. 测试策略

至少补充以下测试：

- 手动开始时能正确写入 `startSource = MANUAL` 和 `manualStartAt`
- 手动结束时能正确写入 `stopSource = MANUAL` 和 `manualStopAt`
- 只有双手动闭环记录段会进入新训练导出
- 自动记录段和旧自动记录数据会被训练导出忽略
- 自动记录入口在 UI 中不再可见
- 过渡期内不会自动启动后台智能记录服务

## 12. 风险与权衡

本设计的主要收益是标签纯度显著提高，但也带来明确权衡：

- 训练数据积累速度会下降
- 用户需要短期内完全手动控制记录
- 自动记录相关代码暂时保留，会造成一段时间内运行时与代码结构不完全一致

这些权衡是可接受的，因为当前阶段的核心目标不是保留自动化体验，而是先建立可靠的人工边界标签闭环。

## 13. 最终结论

本设计采用以下过渡策略：

- 先停用自动记录入口和自动决策触发逻辑
- 保留自动记录底层代码，不做物理删除
- 新训练数据只来自「手动开始 + 手动结束」的完整记录段
- 在现有记录模型中直接记录来源字段与手动边界时间
- 训练导出只保留双手动闭环记录段，并按边界窗口构造 `start / stop` 正样本来源

该方案能以最小架构扰动，为下一阶段模型训练和重新接回自动决策打下干净、稳定的数据基础。
