# 首页 UI 重构设计

日期：2026-04-24
范围：首页记录页视觉重构；历史页与设置页后续跟随统一

## 目标

把当前首页从“厚重功能面板”调整为更接近花瓣地图的“轻盈地图首页”：

- 地图仍然是主角
- 底部信息层更轻、更薄、更精致
- 记录状态更像产品主卡，而不是后台诊断面板
- 底栏导航从“大白盒子”改成更克制的悬浮导航

## 当前问题

基于真机截图，当前首页主要有这些问题：

1. 地图区域很轻，但底部三块白卡体积都很大，视觉重量失衡。
2. “今日里程”“记录状态”“底部 Tab”都使用近似的白色大圆角容器，层级不清晰。
3. 记录状态卡的信息密度高，标签、说明、按钮同时抢注意力，像设置页子面板。
4. 底栏导航高度过高，选中态过厚，遮挡地图的感觉明显。
5. 全页虽然已经偏浅色地图风格，但组件材质语言还不统一。

## 设计方向

采用“花瓣地图轻盈版”。

设计关键词：

- 轻盈
- 留白
- 雾面
- 低对比层级
- 地图优先
- 单主卡

不采用：

- 厚阴影
- 多层大白卡堆叠
- 过重胶囊按钮
- 后台面板式诊断视觉

## 重构方案

### 1. 地图上方状态胶囊

保留左上角 GPS/状态胶囊，但做更轻的花瓣地图式处理：

- 高度略收窄
- 左侧状态点更小
- 主标题与副标题距离更紧
- 箭头更细
- 阴影更淡，边框更轻

目标是像“地图内悬浮提示”，而不是通知卡片。

### 2. 数据摘要条

把“今日里程 / 时长 / 速度”从一整块大卡，改成更薄的横向摘要条：

- 高度显著降低
- 每个指标等宽分布
- 标签更小，数值更紧凑
- 整体做浅雾面，不做厚重白底

这一层只承担“扫一眼看数据”的职责，不再做页面主视觉。

### 3. 记录状态主卡

首页只保留一个主卡，作为下半区的唯一重点内容。

结构：

- 顶部：标题 + 风险状态胶囊
- 中部：一句核心说明
- 列表区：最多保留 1 条最重要的诊断项
- 底部：一个主按钮、一个次按钮

视觉要求：

- 降低卡片高度
- 更大内边距，但更少内容
- 风险标签缩小
- 主按钮保留强调色，但按钮整体更薄
- 次按钮改成更轻的描边按钮

删除“后台诊断页搬到首页”的感觉，只保留最关键的一条风险与入口。

### 4. 底部导航

底部导航改成更薄的悬浮导航：

- 降低容器高度
- 取消过大的选中背景块
- 图标和文字间距收紧
- 选中态更像柔和高亮，而不是独立按钮
- 导航整体向下贴边，减少对地图的遮挡

目标是让它看起来像系统级地图导航，而不是单独一张功能卡。

### 5. 材质统一

首页统一使用一种材质语言：

- 浅暖灰白底
- 很轻的模糊/雾面感
- 很低的描边
- 阴影只做层次提示，不做厚重体积
- 圆角统一到同一组尺度

不再混用“玻璃卡”“液态卡”“厚卡”“大胶囊”四种感觉。

## 页面层级

首页从上到下的视觉优先级调整为：

1. 地图与当前位置
2. 左上状态胶囊
3. 数据摘要条
4. 记录状态主卡
5. 底部导航

这样用户第一眼先看地图，第二眼看状态，第三眼才是操作。

## 实现边界

本次只做记录页首页重构，尽量不改业务逻辑：

- 不改地图交互逻辑
- 不改定位逻辑
- 不改记录状态判断逻辑
- 只重构 Compose 结构、间距、容器、字号、按钮和层级

如果首页效果成立，再把相同语言扩展到历史页和设置页。

## 验收标准

完成后应满足：

1. 首页下半区明显更轻，不再像三层白卡堆叠。
2. 地图主体更突出。
3. 记录状态卡一眼能看懂，不显得杂乱。
4. 底部导航更薄、更稳，不再抢戏。
5. 整体更接近花瓣地图的轻盈感，而不是后台工具页。
