# 首页地图上拉面板重构设计

日期：2026-04-25
范围：首页记录页交互结构与视觉层级；历史页、设置页、地图详情页不在本轮重构范围内。

## 目标

把首页从“地图 + 多层信息卡片”调整为“默认地图沉浸 + 上拉记录面板”：

- 默认状态以地图为主，只保留轻量摘要与底部导航。
- 展开状态承载记录健康、关键指标、主操作和详情入口。
- 地图手势或点击地图空白区域时，面板回到收起状态。
- 始终保持一个最强主操作，避免诊断信息抢占首页主层级。

## 交互设计

### 默认收起状态

- 底部只显示一个轻量摘要面板。
- 摘要面板包含拖拽把手、今日里程、时长、速度和记录状态胶囊。
- 摘要面板支持点击或上拉展开。
- 地图视口底部 padding 跟随摘要面板高度计算，避免当前位置被遮挡。

### 展开状态

- 底部面板扩展为记录操作面板。
- 面板内容顺序：把手、指标摘要、记录健康标题/状态、简短说明、最多一条重点风险、主按钮、详情按钮。
- 完整诊断继续放在弹窗中，不直接铺在首页。
- 下拉超过阈值收起。

### 地图交互

- 用户拖动地图时收起面板，并显示定位按钮强调态。
- 点击地图空白区域时收起面板。
- 点击用户位置点仍打开定位信息弹窗。

## 视觉设计

- 面板使用浅雾面材质，低阴影、低描边、圆角偏大。
- 底部导航变薄、贴底，选中态更轻，减少与上拉面板竞争。
- 定位按钮保持 44dp 以上可点区域，但视觉更轻。
- 指标文字使用数字优先、标签弱化的层级。

## 实现边界

- 修改首页 Compose 布局与地图点击回调。
- 小幅调整设计系统里的底部导航和浮动按钮样式。
- 不改业务状态模型、不改数据库、不改历史页和设置页结构。

## 验证标准

- 首页默认进入时展示轻摘要，而不是只露出把手。
- 上拉或点击摘要可以展开完整记录面板。
- 地图拖动或点击空白区域可以收起面板。
- 定位按钮随面板高度上移，不遮挡摘要或展开面板。
- Kotlin 编译通过。
